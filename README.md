# Capybara Dubai Runner (Name of Game: Приключения Мухаммада в Дубае) 🏙️🦫🍊

Простая мобильная игра-раннер в стиле Subway Surfers. Главный герой — пацанчик верхом на капибаре — катается по Дубаю, собирает апельсины и уворачивается от препятствий. Игра на русском языке

## 🎯 Что нужно сделать

Создай **одностраничную HTML5 игру** с использованием **Three.js** для 3D-графики. Всё должно работать в одном `index.html` файле (или максимум: `index.html` + `game.js` + `style.css`), без сборки, без npm, без бэкенда, без базы данных.

Игра должна нормально работать в мобильном браузере (Chrome/Safari) на телефоне в портретной ориентации.

## 🎮 Геймплей

### Основа
- Бесконечный 3D-раннер от третьего лица (камера сзади-сверху)
- Игрок (пацанчик на капибаре) автоматически бежит вперёд
- 3 дорожки (lanes): левая, центр, правая
- Скорость бега постепенно увеличивается со временем

### Управление (мобильное, через свайпы)
- **Свайп влево/вправо** → перемещение между дорожками
- **Свайп вверх** → прыжок (перепрыгнуть низкое препятствие)
- **Свайп вниз** → подкат / нагнуться (пролезть под высокое препятствие)
- На десктопе: **стрелки** или **WASD** должны делать то же самое (для отладки)

### Сбор предметов
- 🍊 **Апельсины** — основная "валюта", разбросаны по дорожкам, +1 очко за штуку
- Иногда — апельсиновая "цепочка" (ряд из 5-7 апельсинов подряд для дофамина)

### Препятствия (минимум 3 типа)
1. **Низкие** (дорожный конус, ящик) — нужно **перепрыгнуть**
2. **Высокие** (шлагбаум, висящий баннер) — нужно **подкатом / нагнуться**
3. **Широкие** (машина на 1 дорожке) — нужно **сменить дорожку**

При столкновении — Game Over.

### UI
- Сверху слева: счётчик апельсинов 🍊 и пройденная дистанция
- Экран старта: название игры, кнопка "Играть", лучший результат (хранить в `localStorage`)
- Экран Game Over: счёт, лучший результат, кнопка "Заново"

## 🎨 Графика и арт

**Важно:** я сам подготовлю спрайты/модели и положу их в папку `assets/`. На старте используй **простые геометрические заглушки**, чтобы я мог играть и тестировать ещё до того, как загружу свой арт.

### Заглушки на старте:
- **Пацанчик на капибаре**: коричневый прямоугольник (капибара) + маленький розовый куб сверху (пацанчик)
- **Апельсины**: оранжевые сферы
- **Препятствия**: серые кубы / красные параллелепипеды разной высоты
- **Дубай-сцена**:
  - дорога — длинная серая плоскость с разметкой дорожек
  - по бокам — простые "небоскрёбы" из вытянутых параллелепипедов разных размеров и оттенков (бежевый, песочный, серый)
  - вдалеке — силуэт **Бурдж-Халифы** (очень высокий тонкий параллелепипед, чтоб узнавался)
  - небо — градиент от оранжевого к голубому (закат над пустыней)
  - песочно-золотистый туман на горизонте

### Структура папки `assets/`:
```
assets/
  capybara.glb      (3D-модель капибары с пацанчиком)
  orange.png        (текстура апельсина)
  obstacles/
    cone.glb
    barrier.glb
    car.glb
  buildings/
    burj_khalifa.glb
```
Код должен **автоматически использовать модель из `assets/`, если она есть, иначе — заглушку**. Это нужно, чтобы я мог постепенно заменять заглушки на свой арт без переписывания кода.

## 🛠 Технические требования

- **Three.js** через CDN (например, `https://unpkg.com/three@0.160.0/build/three.module.js`)
- Никаких сборщиков (Vite, Webpack и т.д.) — открыл `index.html` и играешь
- Адаптация под мобильный экран: `<meta viewport>`, canvas на весь экран, портретная ориентация
- **Хранение лучшего счёта** — `localStorage` (никаких баз данных)
- 60 FPS на среднем телефоне — следи за производительностью:
  - переиспользуй геометрии и материалы
  - удаляй объекты, которые ушли за камеру
  - не более ~50-100 объектов в сцене одновременно
- Звук — опционально (Web Audio API), не критично
